本服务是多模型 API 网关。用户调用模型接口时，平台会根据用户选择、路由配置、模型可用状态和风控策略，将必要输入数据发送给实际模型供应商进行推理。

## 1. 数据流向概览

```text
用户小程序或 API 客户端
  -> 平台 API 网关
  -> 鉴权、计量、风控、内容安全检查
  -> 实际启用的模型供应商
  -> 平台 API 网关
  -> 用户客户端
```

## 3. 发送给供应商的数据

根据接口调用需要，可能发送：

- 用户输入文本。
- 对话上下文。
- 模型名称。
- 推理参数，例如温度、最大输出长度、是否流式返回。
- 工具调用或函数调用参数。

通常不应发送：

- 用户 API Key 明文。
- 平台内部密钥。
- 支付密钥、证书、Cookie。
- 与本次模型推理无关的订单、发票、退款或客服数据。

## 4. 用户注意事项

用户不应提交：

- 无权处理的个人信息。
- 身份证、银行卡、密码、验证码、Cookie、密钥等敏感信息。
- 商业秘密或未公开代码。
- 违法、侵权、诈骗、攻击或批量滥用内容。

如业务确需处理敏感数据，用户应先确认自己具备合法授权，并采取脱敏、最小化和访问控制措施。

## 5. 供应商变化

平台可能因成本、性能、可用性、安全或合规原因调整模型供应商。重大变化应更新：

- 小程序模型列表。
- 隐私政策。
- 本供应商说明。
- 管理后台合规配置。

成功标志：小程序模型列表中展示的供应商、地区和用途与本说明一致。
