# 待确认字段填空清单

这份清单用于把 `docs/compliance/` 中所有 `【待确认：...】` 字段集中管理。上线前不要直接在多份文档里零散填写，建议先在本表确认，再同步到协议、隐私政策、小程序后台和管理后台。

本表不应填写 API Key、AppSecret、支付密钥、证书正文、Cookie、身份证、银行卡或公司机密。

## 状态说明

| 状态 | 含义 |
| --- | --- |
| 待确认 | 还没有负责人确认 |
| 待技术确认 | 需要技术确认域名、服务器、数据、供应商或系统能力 |
| 待运营确认 | 需要运营确认服务名称、类目、套餐、退款或页面口径 |
| 待客服确认 | 需要客服确认联系方式、服务时间和处理时限 |
| 待法务/合规确认 | 需要律师或合规顾问审阅 |
| 已确认 | 已有最终值，并同步到相关文档和后台 |

## 填写流程

1. 产品/运营先确认服务名称、简介、类目、套餐和退款条件。
2. 技术确认 API 域名、服务器地区、模型供应商、数据保存期限和删除方式。
3. 客服确认电话、邮箱、在线客服、投诉、注销和数据请求入口。
4. 法务/合规审阅协议、隐私政策、内容安全和退款规则。
5. 将最终值同步到小程序后台、管理后台“合规配置”、公开 URL 和代码配置。
6. 最后复查所有 `【待确认：...】` 是否已处理。
