# 合规材料预备包

这套材料用于微信小程序审核、生产上线前内部确认，以及后续律师或合规顾问审阅。当前版本是草案模板，不构成法律意见。

不要在这些文件中填写 API Key、AppSecret、支付密钥、证书正文、Cookie、身份证、银行卡或公司机密。

## 待确认字段

上线前必须确认并替换以下字段：

- `【待确认：服务名称】`
- `【待确认：经营主体全称】`
- `【待确认：统一社会信用代码是否展示】`
- `【待确认：注册地址或联系地址】`
- `【待确认：客服电话】`
- `【待确认：客服邮箱】`
- `【待确认：投诉邮箱或投诉入口】`
- `【待确认：在线客服入口】`
- `【待确认：生产 API 域名】`
- `【待确认：隐私政策 URL】`
- `【待确认：用户服务协议 URL】`
- `【待确认：服务器地区】`
- `【待确认：实际启用模型供应商名单】`
- `【待确认：日志保存期限】`
- `【待确认：业务数据保存期限】`
- `【待确认：退款审核时限】`
- `【待确认：退款到账时限】`

## 建议使用顺序

1. 先复制 [待确认字段填空清单](./fill-in-checklist.md)，明确每个字段的负责人和状态。
2. 产品和运营确认服务范围、套餐规则、退款条件。
3. 技术确认服务器地区、数据保存期限、模型供应商和数据流向。
4. 客服确认电话、邮箱、投诉入口和处理时限。
5. 按 [小程序审核材料包](./miniapp-review-pack.md) 准备微信后台字段、截图和提交说明。
6. 最后交给律师或合规顾问审阅。
7. 审阅通过后，把最终 URL 填入管理后台“合规配置”和微信小程序后台。

## 成功标志

- 小程序内隐私、内容安全、客服、投诉、退款说明口径一致。
- 微信后台隐私保护指引与本材料不冲突。
- 套餐页、订单中心和退款规则不互相矛盾。
- 管理后台“上线检查”无经营主体、协议、客服、供应商披露阻塞项。
