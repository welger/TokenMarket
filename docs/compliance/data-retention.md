本说明用于解释平台保存哪些数据、保存多久、如何删除或匿名化。具体期限上线前需要由技术、运营、财务和合规共同确认。

## 1. 保存期限表

| 数据类型 | 示例 | 建议保存期限 | 删除或处理方式 |
| --- | --- | --- | --- |
| 账号信息 | 微信 openid、账号状态 | `【待确认：业务数据保存期限】` | 注销后删除或匿名化，法律要求保存的除外 |
| API Key 元数据 | Key 名称、脱敏 Key、状态、创建/停用时间 | `【待确认：业务数据保存期限】` | 删除账号后删除或匿名化 |
| API Key 明文 | 创建时展示的完整 Key | 不保存 | 创建后只展示一次 |
| 调用日志 | 请求 ID、模型、字符数、状态码、耗时 | `【待确认：日志保存期限】` | 到期删除或聚合匿名化 |
| 用户输入和输出内容 | prompt、消息、模型返回内容 | `【待确认：是否保存输入输出原文及期限】` | 尽量不保存原文；如保存，到期删除或脱敏 |
| 订单信息 | 订单号、套餐、金额、支付状态 | `【待确认：订单数据保存期限】` | 按财务和争议处理要求保存，到期归档或匿名化 |
| 支付和退款记录 | 微信交易号、退款状态、审核记录 | `【待确认：支付退款数据保存期限】` | 按支付、财务和监管要求保存 |

| 客服投诉记录 | 问题描述、处理过程、联系方式 | `【待确认：客服投诉数据保存期限】` | 争议结束并到期后删除或匿名化 |

## 2. 删除申请

用户可以申请删除个人信息。平台收到申请后，将核验身份和请求范围，并在合理期限内处理。

删除入口：

- 客服邮箱：`【待确认：客服邮箱】`
- 投诉入口：`【待确认：投诉邮箱或投诉入口】`
- 在线客服：`【待确认：在线客服入口】`

以下数据可能无法立即删除：

- 法律法规要求保存的数据。
- 财务、税务、支付、发票和退款所需数据。
- 正在处理投诉、争议、违规、安全事件所需数据。
- 为保护用户或第三方合法权益必须保留的数据。

## 3. 账号注销

用户可以申请注销账号。注销前应确认：

- 已处理未完成订单、退款和发票。
- 已停用或删除不再使用的 API Key。
- 已导出需要自行保存的数据。
- 注销后可能无法恢复账号、订单和用量记录。

账号注销入口见 [客服、投诉、账号注销和数据请求入口说明](./support-and-complaints.md)。

## 4. 数据导出

用户可以申请导出与自身账号相关的订单、套餐、用量和 API Key 元数据。平台不会导出 API Key 明文。

导出方式：`【待确认：数据导出方式】`

## 5. 内部操作要求

- 不在日志中记录完整 API Key、AppSecret、支付密钥、证书正文或 Cookie。
- 不在客服工单中要求用户提供支付密码、短信验证码或银行卡密码。
- 删除或导出请求应保留处理记录，便于审计。
- 对无法删除的数据，应向用户说明原因和预计保存期限。
