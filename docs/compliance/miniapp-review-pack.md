# 小程序审核材料包

这份材料用于微信小程序认证通过后，准备小程序后台提交审核、隐私保护指引、客服入口、服务说明和截图核对。最终字段以微信公众平台后台实际要求为准，本材料不替代律师或合规顾问审核。

不要在本文件填写 API Key、AppSecret、支付密钥、证书正文、Cookie、身份证、银行卡或公司机密。

## 3. 隐私保护指引准备

微信后台隐私保护指引应与 [隐私政策草案](./privacy-policy.md)、[数据保存期限说明](./data-retention.md) 和 [模型供应商说明](./model-provider-disclosure.md) 一致。

建议准备以下口径：

| 后台字段方向 | 建议材料 | 待确认值 |
| --- | --- | --- |
| 开发者/经营主体 | `【待确认：经营主体全称】` | 待法务/合规确认 |
| 收集的信息 | 微信 openid、账号状态、API Key 元数据、调用统计、订单支付、发票、退款、客服投诉、安全风控记录 | 待法务/合规确认 |
| 收集目的 | 登录、鉴权、模型调用、计量计费、套餐发放、订单支付、发票退款、客服投诉、安全风控 | 待法务/合规确认 |
| 共享对象 | `【待确认：实际启用模型供应商名单】`、微信支付、云服务或基础设施服务 | 待技术确认 |
| 服务器地区 | `【待确认：服务器地区】` | 待技术确认 |
| 保存期限 | `【待确认：日志保存期限】`、`【待确认：业务数据保存期限】` | 待技术确认 |
| 删除方式 | 通过客服或数据请求入口申请删除，法律法规要求保存的数据除外 | 待客服确认 |
| 隐私政策 URL | `【待确认：隐私政策 URL】` | 待技术确认 |

提交前自检：

- 隐私保护指引没有遗漏模型供应商。
- 没有声明“不共享数据”后又把输入发送给模型供应商。
- 保存期限与实际日志、订单、发票、退款策略一致。
- 删除方式真实可执行。

## 4. 交易、支付、退款和发票材料

| 项目 | 准备内容 | 状态 |
| --- | --- | --- |
| 套餐说明 | 套餐名称、额度、有效期、价格、适用模型、退款条件 | 待运营确认 |
| 微信支付说明 | 用户在订单中心点击“微信支付”，完成支付后由微信支付通知驱动套餐发放 | 待技术确认 |
| 退款规则 | [退款规则草案](./refund-policy.md) 已替换真实审核时限和到账时限 | 待运营确认 |
| 发票说明 | 用户可在订单中心查看发票记录或按平台流程申请发票 | 待运营确认 |
| 支付回调 | `https://api.example.com/payments/wechat/notify` 替换为 `【待确认：生产支付回调 URL】` | 待技术确认 |

注意：

- 退款说明不得与套餐页冲突。
- 支付成功、套餐发放、退款记录应能在订单中心解释清楚。
- 不在审核材料里填写商户 API v3 Key、商户私钥或平台证书正文。

## 5. 内容安全材料

内容安全说明应与 [内容安全规则草案](./content-safety-rules.md) 一致。

审核口径：

> 本服务禁止用户通过平台提交或生成违法违规、诈骗、攻击、侵权、批量滥用、恶意营销、涉黄涉赌涉毒、侵犯个人信息或商业秘密等内容。平台可对违规请求采取拒绝、限流、停用 API Key、暂停账号、保留证据和依法配合处理等措施。

需要确认：

- 内容安全规则 URL：`【待确认：内容安全规则 URL】`
- 误判申诉入口：`【待确认：投诉邮箱或投诉入口】`
- 后台至少启用一条内容安全规则。

## 7. 提交前截图清单

建议在 375px 和 390px 宽度各保留一组截图：

- 首页：能看到服务介绍和经营主体。
- 模型列表：能看到模型、功能、计费单位、状态。
- 服务套餐：能看到价格、有效期、退款条件。
- API 文档：能看到请求地址、鉴权方式、参数、错误码、示例。
- 密钥管理：能看到“密钥只完整显示一次”的说明或交互。
- 用量统计：能看到调用次数、输入输出量、套餐消耗。
- 订单中心：能看到订单、付款状态、发票和退款入口。
- 隐私说明：能看到供应商、服务器地区、保存期限、删除方式。
- 内容安全：能看到违法内容、诈骗、攻击、侵权、批量滥用禁用规则。
- 客服投诉：能看到真实客服电话或在线客服入口。

## 8. 提交前自检

- `docs/compliance/fill-in-checklist.md` 中所有上线必填项已处理。
- 微信后台隐私保护指引与隐私政策、供应商说明、数据保存说明一致。
- 小程序内页面展示与审核材料一致。
- 客服和投诉入口真实可用。
- 支付、退款和发票规则与套餐页一致。
- 未在任何审核材料中填写密钥、证书正文、Cookie、身份证、银行卡或公司机密。
- 律师或合规顾问已审阅正式版本。
