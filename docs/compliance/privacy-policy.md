## 1. 我们处理的信息

为提供多模型 API 网关和开发接口服务，我们可能处理以下信息：

| 信息类型 | 示例 | 使用目的 |
| --- | --- | --- |
| 账号信息 | 微信 openid、登录状态、账号状态 | 登录、识别用户、账号安全 |
| API Key 信息 | Key 名称、脱敏 Key、状态、创建时间、停用时间 | 密钥管理、鉴权、安全审计 |
| 调用信息 | 请求时间、模型、输入字符数、输出字符数、扣费单位、HTTP 状态、请求 ID、耗时 | 计量计费、用量统计、故障排查 |
| 用户输入数据 | 用户通过 API 发送的 prompt、消息、参数 | 发送给模型供应商进行模型推理 |
| 订单和支付信息 | 订单号、套餐、金额、币种、支付状态、微信交易号、退款状态 | 订单履约、支付核对、退款和发票 |
| 发票信息 | 发票抬头、税号、申请状态 | 发票处理 |
| 客服和投诉信息 | 联系方式、问题描述、处理记录 | 客服支持、投诉处理、争议解决 |
| 安全风控信息 | IP 哈希、请求频率、异常行为记录 | 防止滥用、攻击、欺诈和违规内容 |

我们不会在页面或日志中主动展示 API Key 明文。API Key 创建后只完整显示一次。

实际供应商、用途和地区以 [模型供应商与数据流向说明](./model-provider-disclosure.md) 以及小程序内展示为准。

## 3. 我们如何使用信息

我们使用信息用于：

- 创建和管理账号。
- 提供模型 API 调用、路由、鉴权和结果返回。
- 展示模型列表、套餐、订单、发票、退款和用量统计。
- 完成订单支付、套餐发放、退款审核和争议处理。
- 进行安全风控、内容安全、反欺诈、反攻击和批量滥用识别。
- 排查故障、改进服务质量和统计服务运行情况。
- 履行法律法规、监管要求和平台规则要求。

## 4. 我们如何共享信息

我们不会出售用户个人信息。

在以下场景中，可能共享必要信息：

1. 模型供应商：为完成模型推理，将用户输入数据和必要参数发送给模型供应商。
2. 支付服务：通过微信支付处理订单支付、交易核验和退款相关流程。
3. 云服务或基础设施：为部署、数据库、缓存、日志、监控和安全防护提供必要技术处理。
4. 法律要求：根据法律法规、司法或行政机关要求提供必要信息。

共享时应遵循最小必要原则。

## 6. 用户权利

用户可以依法申请：

- 访问或复制个人信息。
- 更正或补充不准确的信息。
- 删除个人信息。
- 撤回授权或同意。
- 注销账号。
- 获取个人信息处理规则说明。
- 投诉或举报个人信息处理问题。

申请入口见 [客服、投诉、账号注销和数据请求入口说明](./support-and-complaints.md)。

## 7. 安全措施

我们将采取合理安全措施保护信息，包括：

- API Key 哈希保存，明文只展示一次。
- 访问控制和权限隔离。
- 请求 ID 和审计日志用于追踪异常。
- 敏感配置通过环境变量或密钥系统管理。
- 不在代码仓库保存真实密钥、证书或 Cookie。
- 对异常调用、批量滥用和攻击行为进行限制。

## 8. 未成年人

本服务面向开发者和企业/组织使用场景。未成年人使用本服务前，应取得监护人同意。用户不得上传未成年人敏感信息或其他无权处理的信息。

## 9. 政策更新

我们可能根据法律法规、业务变化或供应商变化更新本政策。重大变化将通过小程序页面、公告或其他合理方式提示。
