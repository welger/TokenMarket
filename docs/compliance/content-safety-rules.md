# 内容安全规则草案

服务名称：`【待确认：服务名称】`

经营主体：`【待确认：经营主体全称】`

本规则适用于用户通过小程序、API、控制台或其他入口提交给平台和模型供应商的内容，也适用于用户使用模型输出的行为。

## 1. 禁止内容

用户不得提交、生成、传播或协助传播以下内容：

### 1.1 违法违规内容

- 违反法律法规、监管要求、公序良俗的内容。
- 危害国家安全、公共安全或社会秩序的内容。
- 宣扬恐怖主义、极端主义、暴力犯罪或自残自杀的内容。
- 涉黄、赌博、毒品、违禁品、非法交易等内容。

### 1.2 诈骗和恶意营销

- 仿冒政府、金融机构、平台、客服或他人身份。
- 钓鱼链接、虚假投资、刷单返利、贷款诈骗、中奖诈骗。
- 诱导用户转账、提供验证码、提供密码或泄露账号。
- 批量生成垃圾营销、骚扰信息或引流话术。

### 1.3 攻击和滥用

- 扫描、入侵、攻击、绕过鉴权、提权、窃取数据。
- 生成恶意代码、漏洞利用、木马、钓鱼页面或攻击脚本。
- 绕过平台限流、计费、内容安全或模型安全策略。
- 批量注册、批量调用、倒卖接口或共享账号。

### 1.4 侵权内容

- 未经授权处理或传播他人个人信息。
- 侵犯著作权、商标权、肖像权、名誉权、商业秘密等权益。
- 生成冒充他人的内容，或误导他人以为内容来自特定真实主体。

### 1.5 高风险专业建议

用户不得把模型输出直接作为医疗、法律、金融、投资、安全生产等高风险决策的唯一依据。相关场景应由具备资质的专业人员复核。

## 2. 平台处理措施

平台发现或合理怀疑违规时，可以采取：

- 拒绝请求。
- 限制调用频率。
- 停用 API Key。
- 暂停或终止账号服务。
- 暂停订单履约或退款处理，进行人工核验。
- 保存必要证据。
- 按法律法规要求向主管机关、司法机关或受害方协助处理。

## 3. 用户申诉

如果用户认为平台误判，可以通过以下入口提交申诉：

- 客服邮箱：`【待确认：客服邮箱】`
- 投诉入口：`【待确认：投诉邮箱或投诉入口】`

申诉时建议提供：

- 请求 ID。
- 账号信息。
- 调用时间。
- 被拦截原因截图或错误码。
- 合法使用场景说明。

不要在申诉中提供 API Key 明文、支付密码、短信验证码或 Cookie。

## 4. 管理后台配置要求

上线前应确认：

- 至少一条内容安全规则启用。
- 规则覆盖违法内容、诈骗、攻击、侵权和批量滥用。
- 小程序内容安全页与本规则口径一致。
- 客服和投诉入口可访问。

成功标志：管理后台“上线检查”中内容安全规则无阻塞项。
