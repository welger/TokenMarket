# 合规材料预备包设计

## 目标

在微信小程序认证审核期间，提前准备可审阅、可替换真实信息的合规材料草案。材料用于小程序审核、上线前内部确认、客服与投诉口径统一，不替代律师或合规顾问审核。

## 输出范围

新增 `docs/compliance/` 文档包：

- `README.md`：材料总览、待确认字段、使用顺序。
- `terms-of-service.md`：用户服务协议草案。
- `privacy-policy.md`：隐私政策草案。
- `refund-policy.md`：退款规则草案。
- `data-retention.md`：数据保存期限和删除方式说明。
- `model-provider-disclosure.md`：模型供应商与数据流向说明。
- `content-safety-rules.md`：内容安全规则。
- `support-and-complaints.md`：客服、投诉、账号注销和数据请求入口说明。

同步更新：

- `docs/runbooks/production-launch-checklist.md` 链接合规材料包。

## 写作原则

- 使用“待确认”占位字段，不编造真实经营主体、证照、客服电话或邮箱。
- 不写真实 API Key、AppSecret、支付密钥、证书、Cookie、身份证、银行卡或公司机密。
- 同一概念口径保持一致：经营主体、服务名称、数据保存期限、删除方式、退款条件、客服投诉入口。
- 明确多模型 API 网关服务会把用户输入发送给模型供应商用于模型推理。
- 明确禁止违法内容、诈骗、攻击、侵权和批量滥用。
- 明确用户可以申请访问、更正、删除、导出和注销账号。

## 后续需要真实确认的信息

- 经营主体全称。
- 统一社会信用代码或主体证照信息是否展示。
- 注册地址或联系地址是否展示。
- 真实客服电话、客服邮箱、投诉邮箱或在线客服链接。
- 生产 API 域名、隐私政策 URL、用户协议 URL。
- 实际服务器地区。
- 实际启用的模型供应商、供应商地区、数据处理目的。
- 实际日志、订单、发票、退款、账号数据保存期限。
- 退款审核时限和退款到账时限。

## 验收标准

- `docs/compliance/` 下文档齐全。
- 上线清单能链接到合规材料包。
- 文档没有真实密钥、证书正文、Cookie、身份证、银行卡或公司机密。
- 所有占位字段统一使用 `【待确认：...】` 标注，方便后续搜索替换。
