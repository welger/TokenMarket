# 小程序微信支付接入与上线准备设计

## 目标

在微信认证审核期间，先把已经合并的后端微信支付能力接到小程序订单中心，并补齐生产配置手册。上线时只需要填入真实微信支付配置、完成小额真实支付验收，不需要再临时改代码。

## 范围

- 小程序订单中心识别 `WECHAT + PENDING_PAYMENT` 订单，展示“微信支付”操作。
- 用户点击后调用 `POST /me/orders/:id/pay-wechat` 获取后端签好的 JSAPI 支付参数。
- 小程序调用 `wx.requestPayment` 拉起微信支付。
- 支付成功后刷新订单列表；用户取消或支付失败时展示可理解提示。
- 新增微信支付生产配置手册，说明环境变量、证书文件、回调地址、微信后台配置和上线验收步骤。

不在本次范围内：

- 新建订单购买入口。
- 微信退款 API 自动退款。
- 后端微信支付驱动改造。
- 提交真实商户密钥、证书、AppSecret 或 Cookie。

## 小程序数据流

1. 订单页调用 `loadOrders()` 获取订单列表。
2. `mapOrders()` 把微信待支付订单标记为可支付，付款文案展示“微信支付”。
3. 用户点击“微信支付”按钮。
4. 页面调用 `payWechatOrder(orderId)`。
5. 服务层向 `/me/orders/:id/pay-wechat` 发送 `POST` 请求。
6. 服务层校验返回值包含 `timeStamp`、`nonceStr`、`package`、`signType`、`paySign`。
7. 调用 `wx.requestPayment()`。
8. 成功后刷新订单列表；失败或取消时用 `wx.showModal()` 告知用户。

## 错误处理

- 后端返回错误：沿用现有 HTTP 客户端的公开错误消息，不展示内部错误详情。
- 支付参数缺失：提示“微信支付参数无效，请稍后重试”。
- 用户取消支付：提示“支付未完成，可稍后在订单中心继续支付”。
- 支付失败：提示“微信支付未完成，请稍后重试或联系客服”。
- 支付按钮处理期间禁用重复点击，避免重复拉起支付。

## 测试策略

- `apps/miniapp/test/billing.spec.ts` 覆盖：
  - `WECHAT + PENDING_PAYMENT` 映射为可支付。
  - `payWechatOrder()` 调用正确后端地址并传入 `wx.requestPayment`。
  - 参数缺失时拒绝支付。
  - 微信支付失败时返回统一公开错误。
- 新增或扩展订单页测试，覆盖：
  - 点击微信支付按钮调用支付服务。
  - 支付成功刷新列表。
  - 支付失败展示弹窗。
- 运行 `pnpm --filter miniapp test` 和 `pnpm --filter miniapp typecheck`。

## 上线手册要求

手册只记录配置名和操作步骤，不记录真实值。必须包含：

- `.env`/密钥系统中需要配置的微信支付变量。
- 私钥和平台证书文件的保存建议。
- 微信商户平台和微信公众平台需要配置的项目。
- 回调地址 HTTPS 要求。
- 小额真实订单验收步骤和成功标志。
- 常见失败排查路径。
