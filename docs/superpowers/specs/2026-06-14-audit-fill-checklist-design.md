# 待确认字段填空清单与小程序审核材料包设计

## 目标

在现有 `docs/compliance/` 合规材料草案基础上，新增两份更偏执行的材料：

- 待确认字段填空清单：把所有需要真实确认的信息集中成表格，标注负责人、来源、状态和会影响哪些文档。
- 小程序审核材料包：把微信小程序后台可能要填写、上传、截图或对照的信息提前整理成审核工作台。

## 输出范围

新增：

- `docs/compliance/fill-in-checklist.md`
- `docs/compliance/miniapp-review-pack.md`

修改：

- `docs/compliance/README.md`
- `docs/runbooks/production-launch-checklist.md`

## 填空清单设计

字段表包含：

- 字段名。
- 当前占位符。
- 用途。
- 推荐负责人。
- 最终值来源。
- 出现位置。
- 状态。
- 填写注意事项。

状态统一使用：

- `待确认`
- `待技术确认`
- `待运营确认`
- `待客服确认`
- `待法务/合规确认`
- `已确认`

## 小程序审核材料包设计

审核材料包分为：

- 基础信息：名称、主体、类目、简介、头像。
- 服务说明：多模型 API 网关、套餐、密钥、用量、订单。
- 隐私保护指引：收集信息、使用目的、共享对象、保存期限、删除方式。
- 交易和支付：套餐、微信支付、退款、发票。
- 内容安全：禁止内容、处置措施、申诉入口。
- 客服和投诉：电话、邮箱、在线客服、处理时限。
- 截图和页面路径：小程序内需要截图或核对的页面。
- 提交前自检：上线清单式检查项。

## 安全要求

- 不填写真实身份证、银行卡、密钥、Cookie、证书正文或公司机密。
- 所有真实待填内容继续使用 `【待确认：...】` 占位。
- 明确提醒最终以微信公众平台后台实际字段和律师/合规顾问审阅为准。

## 验收标准

- 两份新文档存在并可从合规 README 进入。
- 生产上线清单的小程序资质部分链接审核材料包。
- 敏感信息扫描无命中。
- 所有真实待填信息均保留 `【待确认：...】`。
