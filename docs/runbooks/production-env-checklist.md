# 生产环境变量部署清单

这份清单用于生产部署前确认 API 服务环境变量、密钥、证书路径、域名、反向代理和验证步骤。不要把真实 API Key、AppSecret、支付密钥、证书正文、Cookie、数据库密码、身份证、银行卡或公司机密写进仓库、截图、PR、Issue 或聊天记录。

## 1. 使用方式

1. 在密钥管理系统或服务器环境变量中配置真实值。
2. 不要把真实值填回 `.env.example` 或本文档。
3. 每项配置完成后，在部署系统或内部表格中记录“已配置/负责人/配置位置”，不要记录密钥明文。
4. 部署后执行本文档的验证步骤。
5. 如验证失败，先回滚流量或保持购买入口关闭，再排查配置。

成功标志：生产 API 可以启动，管理后台“上线检查”无运行环境阻塞项，小程序能登录、创建订单、拉起微信支付并完成小额真实支付验收。

## 2. 基础运行变量

| 变量 | 必填 | 生产要求 | 禁止事项 | 验证方式 |
| --- | --- | --- | --- | --- |
| `NODE_ENV` | 是 | 必须为 `production` | 禁止生产使用 `development` 或 `test` | API 启动日志或进程环境显示 `production` |
| `PORT` | 否 | 默认 `3000`，按部署平台要求设置 | 不要与其他服务端口冲突 | 健康检查能访问 API |
| `DATABASE_URL` | 是 | 指向生产 PostgreSQL，使用独立账号和强密码 | 禁止指向本地库、测试库或把密码提交到 Git | 执行迁移前确认主机、库名和账号 |
| `REDIS_URL` | 是 | 指向生产 Redis 或兼容服务，必要时使用 `rediss://` | 禁止共用本地 Redis 或无认证公网 Redis | 登录、限流和网关调用不报 Redis 连接错误 |

数据库风险提醒：`pnpm --filter api-server prisma migrate deploy` 会修改 `DATABASE_URL` 指向的数据库。执行前必须确认它是生产目标库，并且已经备份。

## 3. 安全密钥变量

| 变量 | 必填 | 生产要求 | 禁止事项 | 验证方式 |
| --- | --- | --- | --- | --- |
| `JWT_ACCESS_SECRET` | 是 | 至少 32 字符，随机生成，只用于当前环境 | 禁止使用 `.env.example` 样例值；禁止复用其他系统密钥 | 用户登录后 token 可用，重启后仍可验证 |
| `API_KEY_PEPPER` | 是 | 至少 32 字符，独立随机值 | 禁止与 JWT、审计、登录限流密钥相同 | API Key 创建和鉴权正常 |
| `AUDIT_IP_HASH_SECRET` | 是 | 至少 32 字符，独立随机值 | 禁止使用真实 IP 明文替代哈希 | 审计日志不出现完整 IP 明文 |
| `ADMIN_LOGIN_THROTTLE_SECRET` | 是 | 至少 32 字符，独立随机值 | 禁止复用 JWT 或 API Key pepper | 管理员登录限流正常 |

轮换提醒：

- `JWT_ACCESS_SECRET` 轮换会影响已签发 token。
- `API_KEY_PEPPER` 轮换会影响现有 API Key 鉴权，必须设计迁移方案。
- 审计和限流密钥轮换会影响历史哈希关联能力。

## 4. 网关限流变量

| 变量 | 必填 | 默认值 | 生产建议 | 验证方式 |
| --- | --- | --- | --- | --- |
| `GATEWAY_IP_RATE_LIMIT_PER_MINUTE` | 否 | `120` | 按真实流量和风控策略设置 | 压测或限流专项测试 |
| `GATEWAY_USER_RATE_LIMIT_PER_MINUTE` | 否 | `60` | 区分普通用户和高配套餐前，先保守设置 | 用户维度限流生效 |
| `GATEWAY_KEY_RATE_LIMIT_PER_MINUTE` | 否 | `60` | 防止单个 API Key 批量滥用 | API Key 维度限流生效 |

成功标志：异常高频调用会被限制，正常小规模调用不受影响。

## 5. 上游模型变量

| 变量 | 必填 | 生产要求 | 禁止事项 | 验证方式 |
| --- | --- | --- | --- | --- |
| `UPSTREAM_BASE_URL` | 是 | 指向实际模型供应商或内部代理的 HTTPS 地址 | 禁止生产使用 `http://127.0.0.1:4010/v1` | 管理后台模型健康检查或一次测试调用成功 |
| `UPSTREAM_API_KEY` | 视供应商而定 | 放在密钥管理系统或服务器环境变量 | 禁止提交到 Git；禁止写入小程序前端 | 真实模型调用通过且日志不输出密钥 |
| `UPSTREAM_DEFAULT_MODEL` | 否 | 配置一个可用默认模型 | 不要填已下线模型 | 未指定模型时调用成功或返回清晰错误 |

供应商披露提醒：启用的供应商必须同步到 [模型供应商与数据流向说明](../compliance/model-provider-disclosure.md) 和小程序模型列表。

## 6. 微信登录变量

| 变量 | 必填 | 生产要求 | 禁止事项 | 验证方式 |
| --- | --- | --- | --- | --- |
| `WECHAT_APP_ID` | 是 | 使用认证通过的小程序 AppID | 禁止使用测试 AppID 上生产 | 真实微信用户能登录 |
| `WECHAT_APP_SECRET` | 是 | 只放服务器环境变量或密钥管理系统 | 禁止写进小程序代码、Git、截图或文档 | `code2session` 登录成功 |
| `WECHAT_TEST_LOGIN_ENABLED` | 是 | 生产必须为 `false` | 禁止生产开启测试登录 | 测试登录入口不可用 |
| `WECHAT_LOGIN_RATE_LIMIT_PER_MINUTE` | 否 | 默认 `30`，按风控策略调整 | 不要设置过高导致刷登录 | 高频登录被限制 |

生产校验规则：`NODE_ENV=production` 时，缺少 `WECHAT_APP_ID`、缺少 `WECHAT_APP_SECRET` 或开启 `WECHAT_TEST_LOGIN_ENABLED=true` 都应阻止启动。

## 7. 微信支付变量

| 变量 | 必填 | 生产要求 | 禁止事项 | 验证方式 |
| --- | --- | --- | --- | --- |
| `PAYMENT_DRIVER` | 是 | 真实支付上线必须为 `wechat` | `NODE_ENV=production` 禁止使用 `test` | 新订单 `paymentDriver` 为 `WECHAT` |
| `WECHAT_PAY_MCH_ID` | 微信支付上线必填 | 小程序已绑定的微信支付商户号 | 禁止填未绑定商户号 | JSAPI 下单成功 |
| `WECHAT_PAY_SERIAL_NO` | 微信支付上线必填 | 商户 API 证书序列号 | 禁止与私钥不匹配 | 微信支付签名通过 |
| `WECHAT_PAY_PRIVATE_KEY_PATH` | 微信支付上线必填 | 指向服务器私钥文件路径 | 禁止提交私钥正文；禁止放项目目录 | 服务进程可读，普通用户不可读 |
| `WECHAT_PAY_PLATFORM_CERT_PATH` | 微信支付上线必填 | 指向微信支付平台证书或平台公钥文件路径 | 禁止提交证书正文到仓库 | 支付通知验签通过 |
| `WECHAT_PAY_API_V3_KEY` | 微信支付上线必填 | 32 字符 API v3 Key，放密钥系统 | 禁止写进文档或 PR | 支付通知 resource 解密成功 |
| `WECHAT_PAY_NOTIFY_URL` | 微信支付上线必填 | 公网 HTTPS，路径为支付通知入口 | 禁止 http；禁止内网地址 | 微信支付回调能到达 `/payments/wechat/notify` |

更多支付配置和验收步骤见 [微信支付生产配置与验收手册](./wechat-pay-production.md)。

成功标志：小额真实订单能完成下单、微信支付、回调验签、resource 解密、金额校验、套餐发放和重复通知幂等处理。

## 8. 反向代理、域名和 HTTPS

| 配置 | 生产要求 | 验证方式 |
| --- | --- | --- |
| `TRUST_PROXY_CIDRS` | 只配置真实反向代理 CIDR；没有代理时留空 | 请求日志和限流使用正确客户端来源 |
| API 域名 | HTTPS，证书有效，加入小程序 request 合法域名 | 小程序真机能请求 API |
| 支付回调域名 | HTTPS，公网可访问，不要求用户登录 | 微信支付通知能到达 |
| 管理后台域名 | HTTPS，限制后台访问和管理员认证 | 管理员登录正常 |
| HSTS/安全头 | 按代理或平台能力配置 | 浏览器检查或安全扫描 |

不要把 `TRUST_PROXY_CIDRS` 配成宽泛公网段。错误信任代理会影响真实 IP 识别、限流和审计。

## 9. 部署前验证命令

在生产部署环境中执行前，确认所有命令连接的是目标生产环境。不要在聊天或日志中输出密钥值。

```bash
node --version
pnpm --version
pnpm --filter api-server prisma migrate deploy
pnpm --filter api-server typecheck
```

启动 API 后验证：

```bash
curl -I https://【待确认：生产 API 域名】/
```

如果生产 API 没有根路径健康检查，可改用实际健康检查路径或反向代理状态检查。

## 10. 部署后人工验收

- 真实微信用户能登录。
- 小程序不允许测试登录。
- 小程序能加载模型列表、套餐、隐私说明、内容安全和客服入口。
- 创建订单后，订单支付方式为微信支付。
- 订单中心能拉起微信支付。
- 小额真实支付后，订单变为已发放，套餐额度到账。
- 管理后台上线检查无阻塞项。
- 日志不出现 API Key、AppSecret、支付密钥、证书正文、Cookie 或用户完整输入内容。
- 监控能看到 API 错误率、P95 延迟、上游失败率、支付回调失败和余额不足。

## 11. 回滚注意事项

如果部署后发现关键配置错误：

1. 先关闭购买入口或切走流量。
2. 保持微信支付回调地址可访问，避免已付款订单无法入账。
3. 不要直接删除订单、支付记录或用户套餐。
4. 修复环境变量后重启服务，并重新执行小额真实支付验收。
5. 对异常订单按微信商户平台交易记录和平台订单记录人工核对。
