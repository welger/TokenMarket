# 生产上线清单

这份清单用于微信小程序认证通过后做上线验收。不要把 API Key、AppSecret、支付密钥、证书私钥、Cookie、身份证、银行卡或公司机密写进仓库。

## 1. 微信小程序资质

- 微信公众平台小程序已完成企业或个体工商户认证。
- 小程序名称、类目、头像、简介与实际服务一致。
- 隐私保护指引已填写，并覆盖模型供应商、服务器地区、保存期限、删除方式。
- 成功标志：微信后台显示认证通过，隐私保护指引已发布。

## 2. 经营主体和客服

- 管理后台“合规配置”填写真实经营主体。
- 填写真实客服电话、在线客服入口或投诉入口。
- 公开页面、小程序内说明和微信后台主体信息不互相矛盾。
- 成功标志：管理后台“上线检查”中经营主体、客服入口、投诉渠道均通过。

## 3. 协议和内容安全

- 隐私政策、用户协议、内容安全规则均有 HTTPS URL。
- 内容安全规则明确禁止违法内容、诈骗、攻击、侵权和批量滥用。
- 至少一条内容安全规则处于启用状态。
- 成功标志：公开 URL 可访问，管理后台“上线检查”中协议和内容安全均通过。

## 4. 模型供应商披露

- 每个启用供应商都填写展示名称、用途和服务器地区。
- 至少一个模型处于可用状态，并绑定启用供应商。
- 隐私说明中写清输入数据会发送给哪些模型供应商。
- 成功标志：小程序模型列表可展示供应商、功能、计费单位和服务状态。

## 5. 微信登录配置

- 生产环境配置 `WECHAT_APP_ID`。
- 生产环境配置 `WECHAT_APP_SECRET`，只放在服务器环境变量或密钥管理系统。
- 生产环境关闭 `WECHAT_TEST_LOGIN_ENABLED`。
- 成功标志：真实微信用户能登录，小程序测试登录入口不可用。

## 6. 微信支付准备

- 微信支付商户号已开通并与小程序 AppID 关联。
- API v3 Key、商户证书序列号、商户私钥路径、支付通知 URL 已配置到生产密钥系统。
- 支付回调 URL 使用 HTTPS，且公网可访问。
- 退款流程、退款条件和发票说明已在套餐和订单中心展示。
- 成功标志：小额真实订单能完成下单、支付回调、套餐发放和退款测试。

## 7. 域名和 HTTPS

- 小程序 request 合法域名已配置到微信后台。
- API 域名、管理后台域名、支付回调域名均启用 HTTPS。
- 反向代理只信任真实代理 CIDR，不使用宽泛来源。
- 成功标志：`NODE_ENV=production` 时管理后台“上线检查”没有运行环境阻塞项。

## 8. 数据库、缓存和备份

- PostgreSQL 和 Redis 使用生产独立实例。
- 数据库账号按最小权限配置。
- 每日备份已启用，且至少做过一次恢复验证。
- 成功标志：能从备份恢复到临时库，并完成登录、订单和用量数据抽查。

## 9. 监控和日志

- API 错误率、P95 延迟、上游失败率、支付回调失败、余额不足、内容拦截都有监控。
- 日志不记录 API Key、AppSecret、支付密钥、用户完整输入内容。
- 关键错误有告警联系人。
- 成功标志：人工触发一次测试告警，负责人能收到。

## 10. 发布前最终确认

- 管理后台“上线检查”无阻塞项。
- CI 测试通过。
- 小程序 375px / 390px 核心页面截图验收通过。
- 生产环境变量只在服务器或密钥管理系统中配置，没有提交进 Git。
- 完成一笔真实小额订单的全链路验证。
